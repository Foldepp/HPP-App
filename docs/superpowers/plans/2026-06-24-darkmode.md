# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Dark Mode mit 3-Zustands-Umschalter (Auto/Hell/Dunkel), System-Default, kein Flackern beim Laden.

**Architecture:** Die App nutzt durchgängig CSS-Variablen. Dark Mode = (1) zweite Palette als `:root[data-theme="dark"]`-Override, (2) Audit der noch hartkodierten Farben auf neue Variablen, (3) reine Theme-Logik in `logic.js` (node-getestet), (4) ein DOM-Controller `theme.js` der „auto" via `matchMedia` auflöst und `data-theme` + `theme-color` setzt, (5) winziges Inline-Skript im `<head>` gegen Flackern, (6) Umschalt-Knopf in der Kopfzeile.

**Tech Stack:** Vanilla HTML/CSS/JS (kein Build), `node:test`, localStorage, `matchMedia`, Service Worker (Cache-Bump).

## Global Constraints

- Oberfläche **Deutsch**.
- Vanilla JS, kein Framework, offline lauffähig (`file://`-tauglich bleibt nicht Pflicht, App läuft via SW).
- **Gürtelfarben** (`--g-gelb/gruen/blau/braun/schwarz`) bleiben **unverändert** (Marke).
- `app/data.js` ist generiert — nicht anfassen.
- Funktionsnamen der Theme-Logik **exakt** wie in der Spec: `themaAufgeloest(pref, systemDark)`, `naechstesThema(aktuell)`.
- `pref`-Werte sind die deutschen Strings `"auto" | "hell" | "dunkel"`; `data-theme`-Attribut ist `"light" | "dark"`.
- localStorage-Key: `hpp_theme`.
- Bei jeder Shell-Änderung: `CACHE_NAME` in `app/sw.js` hochzählen.

### Task 3: Theme-Controller, No-Flicker-Skript, Umschalter & Service Worker

DOM-Anbindung: `theme.js` löst auf und setzt `data-theme`/`theme-color`; Inline-Skript verhindert Flackern; Knopf in `.kopf`; SW cached `theme.js`.

**Files:**
- Create: `app/theme.js`
- Modify: `app/index.html` (Inline-Skript im `<head>`, `<script src="theme.js">`)
- Modify: `app/app.js` (Umschalt-Knopf in `.kopf` + Wiring)
- Modify: `app/sw.js` (SHELL + `CACHE_NAME`)

**Interfaces:**
- Consumes: `window.HPP_LOGIC.themaAufgeloest`, `window.HPP_LOGIC.naechstesThema` (Task 1).
- Produces: `window.HPP_THEME = { pref(): string, anwenden(): void, umschalten(): string }`. `app.js` ruft `umschalten()` und liest `pref()`.

- [ ] **Step 1: `app/theme.js` anlegen**

```js
(function (root) {
  "use strict";
  var KEY = "hpp_theme";
  var META = { hell: "#f7f6f3", dunkel: "#15181c" };
  var L = root.HPP_LOGIC;

  function pref() {
    try { return localStorage.getItem(KEY) || "auto"; } catch (e) { return "auto"; }
  }
  function systemDunkel() {
    return !!(root.matchMedia && root.matchMedia("(prefers-color-scheme: dark)").matches);
  }
  function anwenden() {
    var thema = L.themaAufgeloest(pref(), systemDunkel());
    document.documentElement.setAttribute("data-theme", thema === "dunkel" ? "dark" : "light");
    var m = document.querySelector('meta[name="theme-color"]');
    if (m) m.setAttribute("content", META[thema]);
  }
  function umschalten() {
    var neu = L.naechstesThema(pref());
    try { localStorage.setItem(KEY, neu); } catch (e) {}
    anwenden();
    return neu;
  }

  if (root.matchMedia) {
    var mq = root.matchMedia("(prefers-color-scheme: dark)");
    var onChange = function () { if (pref() === "auto") anwenden(); };
    if (mq.addEventListener) mq.addEventListener("change", onChange);
    else if (mq.addListener) mq.addListener(onChange);
  }

  anwenden();
  root.HPP_THEME = { pref: pref, anwenden: anwenden, umschalten: umschalten };
})(window);
```
